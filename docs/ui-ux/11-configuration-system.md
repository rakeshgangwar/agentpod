## Configuration Overview

### Configuration Types

| Config Type | Scope | Who Sets | When Modified |
|-------------|-------|----------|---------------|
| **App Settings** | Device | User | Settings page |
| **User OpenCode Settings** | All projects | User | Settings page |
| **Provider Credentials** | All projects | User | Settings or onboarding |
| **Container Tier** | Per project | User or Agent | Project creation/settings |
| **Project Settings** | Per project | User or Agent | Project settings |
| **Project OpenCode Config** | Per project | Agent (via git) | During development |
| **Agents/Commands/Tools** | User or project | User or Agent | Settings or onboarding |

## Configuration Hierarchy

### Priority Order (Highest to Lowest)

1. **Project `.opencode/` config** - Agent-managed, in git
2. **User OpenCode settings** - User-managed via app
3. **Platform defaults** - Baked into Docker image

### What Each Layer Controls

#### User OpenCode Settings (Global)

```typescript
interface UserOpencodeSettings {
  // Model preferences
  model?: string;               // "anthropic/claude-sonnet-4-5"
  small_model?: string;         // "anthropic/claude-haiku-4-5"
  
  // Permissions (global defaults)
  permission?: PermissionSettings;
  
  // Global instructions (applied to all projects)
  instructions?: string[];
  
  // User-defined extensions
  agent?: Record<string, AgentConfig>;
  command?: Record<string, CommandConfig>;
  tools?: Record<string, boolean>;
  mcp?: Record<string, MCPConfig>;
  
  // Provider overrides
  provider?: Record<string, ProviderConfig>;
  disabled_providers?: string[];
}
```

#### Project Settings

```typescript
interface ProjectSettings {
  // Identity
  name: string;
  description?: string;
  
  // Container
  containerTierId: string;      // 'lite' | 'standard' | 'pro' | 'desktop'
  containerVersion: string;
  
  // LLM (overrides user default)
  llmProvider?: string;
  llmModel?: string;
  
  // GitHub Sync
  githubRepoUrl?: string;
  githubSyncEnabled: boolean;
  githubSyncDirection?: 'push' | 'pull' | 'bidirectional';
}
```

### Container Tier Selector

Visual selector for choosing container resources.

```typescript
interface ContainerTierSelectorProps {
  /** Currently selected tier */
  value: string;
  
  /** Available tiers */
  tiers: ContainerTier[];
  
  /** Change handler */
  onChange: (tierId: string) => void;

  showDetails?: boolean;

  disabled?: boolean;

  id: string;           // 'lite', 'standard', 'pro', 'desktop'
  name: string;
  description: string;

  isDefault: boolean;
}
```

### LLM Provider Config

Configure API keys and select default providers.

```typescript
interface LLMProviderConfigProps {
  /** List of available providers */
  providers: ProviderWithModels[];
  
  /** Currently configured provider IDs */
  configuredProviders: string[];
  
  /** Default provider ID */
  defaultProviderId?: string;
  
  /** Configure provider handler */
  onConfigure: (providerId: string, apiKey: string) => Promise<void>;
  
  /** Set default handler */
  onSetDefault: (providerId: string) => Promise<void>;
  
  /** Remove credentials handler */
  onRemove: (providerId: string) => Promise<void>;
  
  /** Show all providers or just popular */
  showAll?: boolean;
}

interface ProviderWithModels {
  id: string;
  name: string;
  logoUrl: string;
  authType: 'api_key' | 'oauth' | 'device_flow';
  apiKeyEnvVar?: string;
  isConfigured: boolean;
  isDefault: boolean;
  models: ModelInfo[];
}
```

## Project Onboarding Agent

### Concept

Instead of a traditional form-based project creation wizard, we use an **AI-assisted onboarding agent** that:

1. **Gathers requirements** through natural conversation
2. **Recommends configuration** based on project type
3. **Sets up the container** with appropriate tier and settings
4. **Initializes the project** with custom agents, commands, and config
5. **Installs dependencies** and configures the development environment

```typescript
interface OnboardingAgentProps {
  /** Callback when project is created */
  onComplete: (project: Project) => void;
  
  /** Callback to cancel */
  onCancel: () => void;
  
  /** Existing GitHub repo URL (if importing) */
  importUrl?: string;
  
  /** User's configured providers */
  providers: ProviderWithModels[];
  
  /** Available container tiers */
  tiers: ContainerTier[];
}

interface OnboardingState {
  phase: 'gather' | 'recommend' | 'customize' | 'creating' | 'complete' | 'error';
  
  // Gathered info
  projectType?: string;
  languages?: string[];
  frameworks?: string[];
  description?: string;
  teamSize?: 'solo' | 'small' | 'large';
  hasExistingRepo?: boolean;
  repoUrl?: string;
  specialRequirements?: string[];
  
  // Recommendations
  recommendations?: OnboardingRecommendations;
  
  // Final config
  finalConfig?: ProjectConfig;
  
  // Progress
  progress?: {
    step: string;
    percent: number;
    details?: string;
  };
}

interface OnboardingRecommendations {
  tier: {
    id: string;
    reason: string;
  };
  
  model: {
    providerId: string;
    modelId: string;
    reason: string;
  };
  
  agents: {
    name: string;
    description: string;
    reason: string;
  }[];
  
  commands: {
    name: string;
    description: string;
    reason: string;
  }[];
  
  environment: {
    runtime: string;           // 'node-20', 'python-3.12', etc.
    packages: string[];        // npm/pip packages to install
    tools: string[];           // CLI tools to install
    reason: string;
  };
  
  permissions: {
    bash: 'allow' | 'ask' | 'deny';
    edit: 'allow' | 'ask' | 'deny';
    write: 'allow' | 'ask' | 'deny';
    reason: string;
  };
}
```

|----------|--------|-------------|

| `/api/users/:id/opencode/mcp` | GET/POST | Manage MCP servers |

### Onboarding Agent Backend

The onboarding agent is a specialized AI agent that:

1. **Runs on the Management API** (not in a container)
2. **Uses a structured conversation** to gather requirements
3. **Has access to project templates** for common setups
4. **Generates recommendations** based on project type
5. **Creates the project** with all configurations

```typescript
// Backend onboarding agent interface
interface OnboardingAgent {
  // Start new onboarding session
  start(userId: string): Promise<OnboardingSession>;
  
  // Process user message
  processMessage(sessionId: string, message: string): Promise<OnboardingResponse>;
  
  // Get recommendations
  getRecommendations(sessionId: string): Promise<OnboardingRecommendations>;
  
  // Create project with config
  createProject(sessionId: string, config: ProjectConfig): Promise<Project>;
}

interface OnboardingResponse {
  message: string;
  quickOptions?: QuickOption[];
  phase: 'gather' | 'recommend' | 'confirm';
  recommendations?: OnboardingRecommendations;
}
```
