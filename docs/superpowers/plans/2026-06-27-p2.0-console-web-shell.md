# P2.0 — Console Web-Shell Port Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development. Steps use checkbox (`- [ ]`) syntax.

**Goal:** Make the Svelte web console usable end-to-end in a browser — connect → login → `/nodes` → adopt → station panels — by porting the app **shell** (connection, auth-persistence, layout init, station-client URL) off **Tauri** to fetch. The new station pages are already fetch-based; only the surrounding shell still calls `@tauri-apps/api/core` `invoke`, which is `undefined` in the browser.

**Architecture:** The console is a SvelteKit SPA (`adapter-static`, `ssr=false`). Auth is Better Auth (the `better-auth/svelte` client is already fetch-based; sessions are cookies). The hub is at a runtime `apiUrl` the user enters on the connect screen. We replace the Tauri connection/auth-storage calls with fetch + `localStorage`, and point the station client at the same runtime `apiUrl`.

**Tech Stack:** SvelteKit + Svelte 5 runes, `better-auth/svelte`, fetch, `localStorage`. Hub: Bun + Hono + Better Auth.

## Global Constraints

- **Branch:** `redesign/fleet-console`. Frequent commits.
- **Scope = the shell + the station flow only.** Do NOT port the legacy 107-function `apps/console/src/lib/api/tauri.ts` or the OpenCode-era pages (`/projects`, `/workflows`, `/admin`, `/settings`) — those are being retired and may stay broken in web. Only `connect/login/layout/auth` + the `/nodes` station flow must work in a browser.
- **No Tauri at module-load or in the shell's runtime path.** Tauri-only code (e.g. `tauri-plugin-mcp`) must be dynamically imported AND guarded so it never runs / throws in a browser.
- **Hub URL is runtime:** the console targets the `apiUrl` entered on the connect screen (persisted in `localStorage`), defaulting to `PUBLIC_HUB_URL ?? http://localhost:3001`. Auth client and station client use the SAME url.
- **Auth = cookie session.** Rely on the Better Auth session cookie (console `:1420` → hub `:3001` is same-site, different port → SameSite=Lax cookies are sent; hub CORS already allows `:1420` with credentials). Drop bearer-token-into-Tauri-storage.
- TDD where it fits (stores have vitest; the visual task is manual E2E). Tests on existing console vitest infra (`$lib` alias, jsdom, `resolve.conditions:["browser"]`).

**Interfaces:**
- Consumes: the Better Auth client (`createAuthClient`, baseURL set by `setAuthApiUrl` from Task 1).

Remove the `$lib/api/tauri` auth imports.
